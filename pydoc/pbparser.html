pbparser Module¶

synopsis:Parsing functions for protocol buffers.

This is created to be our interface to the google-generated parser scripts in vOutput_pb2.

Any required libraries are stored with reef (in the google module).

pbparser.scan_output(path_to_output)¶

Parse output from a vazels experiment.

This is the main entry point to the scanner.

Parameters:
  • path_to_output (str) – The path to an existing Output_Folder inside a vazels experiment directory.
Returns:

A dictionary with the same hierarchy structure as Vazels outputs into the Output_Folder or None if there was an error.
Return type:

dict

If you do not provide the path to an Output_Folder, bad things might happen.

pbparser.scan_output_helper(path)¶

Parse the Output_Folder from a Vazels experiment.

This will recursively scan down through the directories looking for the files at the bottom, continually building dictionaries as it goes to store each layer in. Uses the current folder’s name as the key for the dictionary. This ensures keys will not clash.

Parameters:
  • path (str) – Path to the Output_Folder from a Vazels experiment.
Returns:

A nested dictionary containing the experiment data.
Return type:

dict
Raises :

Absolutely anything - all exceptions are left to propagate.
pbparser.process_file(filepath)¶

Generate a dictionary containing useable data from a buffered protocol file.

Parameters:
  • filepath (str) – Path to the protobuf file.
Returns:

A dictionary containing the data represented by the protobuf file.
Return type:

dict
