handler.settings Module¶

synopsis:Quick and easy way to manage simple settings.
class handler.settings.SettingsEditor¶

Bases: handler.dochandler.DocHandler

Deals with any getting and setting of config files.

Unfortunately this does not deal with validation or anything other than str, int or float variables. This is why it’s not really used anywhere in the application.

GET(request)¶

Get setting data from the config.

Parameters are expected to be part of the query string.

If the base of this address handler is called:

Returns:A list of sections in the config
Return type:list

If anything more than the base of this handler is called then the excess is assumed to be a section name and:

Returns:A JSON representation of that particular section of the config.
Return type:json
Raises :restlite.Status 404 if the section does not exist.
POST(request, entity)¶

Update a section of the settings.

Any part of the requested address after the handler’s base address will be taken as a section name inside the settings. Any parameters are assumed to be in the POST entity and url encoded.

A request can take any number of the following parameters:

To set a setting:

Parameters:
  • <setname>_<type> – The value to set this setting to.

where <type> is one of:

  • "s" for string
  • "i" for integer
  • "d" for double

and <setname> is the name of the setting to set.

To remove a setting:

Parameters:
  • <setname>_r – No value

Only the first occurence of each <setname> is observed.

Returns:

JSON representation of the settings section.
Return type:

json
Raises :

restlite.Status 400 if there is a parameter with an invalid action in the name (the bit after the "_").

restlite.Status 400 if the value of a parameter cannot be parsed with the type given.
handler.settings.splitName(name)¶

Get a parameter name and a letter indicating an action.

  • If name ends in _. where . is any character, return (name, func)
  • If there is no _ give full name as name and None type.
Parameters:
  • name (str) – The name of a parameter in the request.
Returns:

A tuple consisting of the parameter name and action character.
Return type:

(str, str)

Get a parse function corresponding to an action character.

Parameters:
  • ext (str) – The action character.
Returns:

A parser function or None if there is not one matching.
Return type:

func
handler.settings.getSection(path)¶

Pull out the section name from the path.

This means anything after the first “/”.

Parameters:
  • path (str) – The requested path.
Returns:

A section name (possibly non-existent in the settings).
Return type:

str
