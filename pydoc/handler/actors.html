handler.actors Module¶

synopsis:Handles most actor related requests.
handler.actors.ALLOWED_TYPES¶

The list of allowed types for uploading.

class handler.actors.ActorsHandler¶

Bases: handler.dochandler.DocHandler

Documented handler for dealing with actors.

GET(request)¶

Get either the list of actors or actor details.

If the base address of the handler is requested:

Returns:List of actors on the server.

If the base / actorname is requested:

Returns:JSON representation of the actor called actorname
Raises :restlite.Status 404 if the actor does not exist.

If the base / actorname.tar.gz is requested:

Returns:

The contents of the actor file.
Raises :

restlite.Status 404 if the actor does not exist.

restlite.Status 410 if the actor exists but the file is missing.

restlite.Status 500 if the actor file could not be read.
POST(request, entity)¶

Create a new actor.

You should POST to the base address of the handler although this may not be checked. Also the POST should be encoded as mutlipart form data.

The parameters here should be in the POST entity:

Parameters:
  • actor_name (str) – The name of the actor to add.
  • actor_file (file - .tar.gz) – The actor’s actual file.
  • actor_type (str) – The type of the actor. This should be one of the values in ALLOWED_TYPES.
Raises :

restlite.Status 400 if the data cannot be parsed.

restlite.Status 400 if any of the parameters are missing.

restlite.Status 400 if the actor type is invalid.

restlite.Status 403 if overwriting actors is disabled and you try to use an existing actor name.
