config Module¶

synopsis:Simple persistent storage for the reef backend.

This is (possibly) maldesigned as it can only be used once in any application.

config.getSettings(type, create=True)¶

Grab the settings dict for the given type.

Parameters:
  • type (str) – The section (or type) of settings to grab.
  • create (bool) – If type does not exist do we create it or explode?
Returns:

A dictionary containing the settings for the given section.
Return type:

dict
Raises :

KeyError If the type does not exist and is not set to be created.
config.hasSettings(type)¶

Check if the given section exists in the settings.

Parameters:
  • type (str) – The settings section.
Returns:

True if and only if the section already exists.
Return type:

bool
config.getSections()¶

Grab a list of currently available settings sections.

Returns:A list of settings sections.
Return type:list of str
config.setConfig(cfgpath)¶

Either load a config from the given path or create a new one there.

After running this method the path will be saved for other functions like saveConfig().

Parameters:
  • cfgpath (str) – The path to the config file.
config.saveConfig()¶

Save the config file if a location has been set with setConfig().

config.onDisk()¶

Check if the config is currently on the disk.

This is for example if we have used setConfig() to set a file but do not know whether it was there before or not.

Returns:True if and only if a location has been set and a file exists there.
Return type:bool
